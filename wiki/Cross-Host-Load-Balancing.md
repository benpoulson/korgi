# Cross-Host Load Balancing

Korgi uses Traefik as the load balancer, with two routing mechanisms:

1. **Docker provider** -- Traefik discovers containers on its own host via the Docker socket
2. **File provider** -- Korgi generates a dynamic YAML config listing all backends across all hosts

## How it works

After every `deploy`, `scale`, `rollback`, or `destroy`, korgi:

1. Queries the current container state across all hosts
2. Generates a Traefik dynamic config mapping services to `internal_ip:host_port` backends
3. Writes the config into the Traefik container at `/etc/korgi/dynamic.yml`
4. Traefik watches the file and updates routing automatically

## Generated config example

```yaml
# Generated by korgi -- do not edit manually
http:
  routers:
    myapp-api:
      rule: "Host(`api.example.com`)"
      service: myapp-api
      entryPoints:
        - websecure
      tls:
        certResolver: letsencrypt
  services:
    myapp-api:
      loadBalancer:
        servers:
          - url: "http://10.0.0.10:9001"
          - url: "http://10.0.0.11:9002"
          - url: "http://10.0.0.10:9003"
        healthCheck:
          path: /health
          interval: 5s
          timeout: 3s
```

## Host addressing

Each host has two addresses:

| Field | Purpose |
|-------|---------|
| `address` | SSH connections (public/external IP) |
| `internal_address` | Traefik routing (private/internal IP) |

If `internal_address` is not set, `address` is used for both.

```toml
[[hosts]]
name = "worker-1"
address = "203.0.113.10"        # public -- SSH connects here
internal_address = "10.0.0.10"  # private -- Traefik routes here
```

## Port allocation

Containers bind to host ports so Traefik can reach them across the network:

```toml
[services.ports]
container = 8080       # port inside the container
host_base = 9001       # instance 0 -> 9001, instance 1 -> 9002, ...
```

| Field | Description |
|-------|-------------|
| `container` | Port the app listens on inside the container. |
| `host` | Fixed host port (only for single replica per host). |
| `host_base` | Base port for auto-allocation. Each replica gets `host_base + instance_index`. |

## Dedicated load balancer pattern

The recommended setup has a dedicated LB host that only runs Traefik:

```toml
[[hosts]]
name = "lb"
role = "lb"              # Traefik only, no app containers
address = "203.0.113.1"
internal_address = "10.0.0.1"

[[hosts]]
name = "worker-1"
address = "10.0.0.10"
internal_address = "10.0.0.10"
labels = ["app"]

[[hosts]]
name = "worker-2"
address = "10.0.0.11"
internal_address = "10.0.0.11"
labels = ["app"]

[[services]]
name = "api"
placement_labels = ["app"]  # only on workers, not on lb
```

## Co-located mode

If your LB host also has matching placement labels, it runs containers too:

```toml
[[hosts]]
name = "server-1"
role = "lb"
labels = ["app"]  # Traefik AND containers on the same host
```
